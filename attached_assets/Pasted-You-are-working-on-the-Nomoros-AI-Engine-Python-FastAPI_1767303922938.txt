Create a demo-safe, deterministic compliance ingest + summary endpoint for:

- Source of Funds (SoF) evidence (PDF)

3) Endpoint must never fail a demo because the LLM failed. If LLM errors, return a deterministic fallback summary.
4) Return structured JSON for the UI.

D) Deterministic statuses + eligibility (NO LLM)
============================================================

Implement enums:
Provider: thirdfort | credas | other
CheckType: ID_AML | SOF | BOTH
DeterministicStatus: NOT_STARTED | PENDING | PASSED | FAILED | REVIEW_REQUIRED

ProviderStatusPayload model (optional input):
- provider_reference_id: str | None
- id_aml_status: DeterministicStatus | None
- sof_status: DeterministicStatus | None
- flags: list[str] default []
- red_flags_found: bool | None
- last_updated_at: str | None

DeterministicComplianceState output:
- matter_id, provider, provider_reference_id
- id_aml_status, sof_status
- red_flags_found
- provider_flags
- missing_items
- computed_at (UTC ISO)

Rules:
1) If provider_status_json contains id_aml_status/sof_status, use them.
2) If a status is missing but the check_type expects it, set it to REVIEW_REQUIRED.
3) red_flags_found:
   - if payload.red_flags_found is present, use it
   - else red_flags_found = (len(flags) > 0)
4) missing_items must list clear reasons:
   - "Provider AML/ID report missing (or structured provider status not supplied)." when applicable
   - "SoF evidence documents missing." when applicable
   - "ID/AML not cleared (status: X)." when status != PASSED
   - "Source of Funds not verified (status: Y)." when status != PASSED
   - "Provider reported red flags — solicitor review required." when red_flags_found==true

Gate eligibility (deterministic):
- aml = (id_aml_status == PASSED) AND (red_flags_found == false)
- sof = (sof_status == PASSED) AND (red_flags_found == false)
